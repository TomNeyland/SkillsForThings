# Admitting a Skill

![A clean grep can still miss a private project leak. Run deterministic machine checks, give the complete reusable AI instruction package to two independent readers, keep every concrete issue either reader finds, and repeat after substantive fixes until all three checks are clean.](assets/admitting-a-skill.png)

Public release needs two kinds of evidence. Deterministic checks catch malformed structure, local paths, exact references, and size limits. Independent full-package readers catch origin-specific assumptions, coined language, and examples that only make sense inside the project that produced them.

The release rule is simple: machine checks pass, reader A is clean, and reader B is clean. A real issue from either reader returns the package to repair and review.
